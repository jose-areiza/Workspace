## commit

*commit = guardar cambios localmente

Cuando haces cambios en tu código, commit guarda una versión del proyecto en tu computadora, pero todavía no se sube a GitHub.

Ejemplo:

git commit -m "arreglé el login"

✔ Guarda un punto en el historial del proyecto.*

## Push

*push = subir los cambios a GitHub

Después de hacer commit, push envía esos cambios al repositorio en GitHub.

git push

✔ Tus cambios ya quedan en la nube y otros pueden verlos.*

## Pull

*pull = traer cambios de GitHub a tu computadora

Si alguien más cambió el código en GitHub, pull descarga esos cambios a tu proyecto local.

git pull

✔ Mantiene tu proyecto actualizado.*
